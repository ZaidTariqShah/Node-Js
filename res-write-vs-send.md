res.write() is a low-level Node.js method used to send data in chunks while keeping the response open. Once res.write() is called, the response starts streaming and the headers are usually sent automatically. After headers are sent, they cannot be changed anymore. res.send() is a high-level Express method that automatically sets headers, detects content type, converts objects to JSON if needed, sends the response, and ends it. If we use res.write() first and then res.send(), an error can occur because res.send() still tries to set headers internally, but the headers were already sent when res.write() started the response stream. That is why after using res.write(), we usually finish the response using res.end() instead of res.send().
